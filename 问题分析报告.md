# 舌镜项目问题分析报告

## 一、问题根因分析

### 问题1：没有辨证病机的情况

**Bot Prompt状态**：✅ 已有病机要求
- Bot Prompt 中明确要求输出 `pathogenesis: "病机分析"`
- API测试显示Bot实际返回了完整的病机内容

**根因**：前端显示逻辑对病机格式有假设
- `DiagnosisResultDisplay.tsx` 中用 `→` 符号分割病机进行展示
- 如果Bot返回的病机没有 `→` 符号，展示会以整段显示，可能用户感知不明显
- 实际测试：Bot返回的病机是多行文字描述，不含 `→` 符号

### 问题2：配穴显示"待确认"

**双重根因**：

#### 根因A - 穴位映射表不完整
以下常用穴位未在 `ACUPOINT_MERIDIAN_MAP` 中：
- `安眠`（经外奇穴）
- `劳宫`（手厥阴心包经）
- `行间`（足厥阴肝经）

#### 根因B - 配穴解析逻辑缺陷
Bot返回的配穴格式：
```
配穴：
心烦明显：加劳宫
易怒明显：加行间
失眠严重：加百会、申脉
```

前端解析逻辑：
```typescript
const secondaryPointsPatterns = [
  /配穴[：:]\s*([^\n]+)/,  // 只匹配第一行
];
// 结果：'心烦明显：加劳宫'
// 按'、'分割后：['心烦明显：加劳宫']
// 传入 getMeridian('心烦明显：加劳宫') => '待确认'
```

### 修复2：修复配穴解析逻辑

修改 `src/services/api.ts` 中的 `parseMarkdownDiagnosis` 函数，增强配穴解析：

```typescript
// 解析配穴 - 支持条件格式
function parseSecondaryPoints(markdown: string): AcupuncturePoint[] {
  // 查找配穴区域
  const match = markdown.match(/配穴[：:]\s*([\s\S]*?)(?=\n刺法|##|$)/);
  if (!match) return [];
  
  const section = match[1];
  const points: string[] = [];
  
  // 提取所有 "加XXX" 格式
  const addPointMatches = section.match(/加([^\s\d，,。]+)/g);
  if (addPointMatches) {
    addPointMatches.forEach(m => points.push(m.replace('加', '')));
  }
  
  // 去重并过滤空值
  return [...new Set(points)].filter(p => p && !p.includes('无'))
    .map(point => ({ 
      point, 
      meridian: getMeridian(point), 
      effect: getEffect(point), 
      technique: '平补平泻' 
    }));
}
```
